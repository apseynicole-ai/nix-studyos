# New Module Checklist

A quick reference for starting every new module. Tick off in order.

---

## Phase 1 — Intake

- [ ] Module code, full name, semester, year confirmed
- [ ] Module type identified: Law / Quantitative / Mixed
- [ ] All available sources downloaded and in `01_sources/raw/`
- [ ] Module folder structure created (matches system-config.md)
- [ ] `00_intake/module-register.md` completed (Module 01)
- [ ] `01_sources/source-audit.md` completed (Module 02)
- [ ] Coverage gaps identified and assessed

## Phase 2 — Build

- [ ] `00_intake/build-brief.md` approved (Module 03)
- [ ] `02_builds/[MODULE]_S[N]_v0_8_*.html` produced (Module 04)
- [ ] QA Audit 1 completed (Module 06)
- [ ] Patches applied as needed (Module 05)
- [ ] QA Audit 2 completed (Module 06)
- [ ] No P1 or P2 issues outstanding

## Phase 3 — Print Preparation

- [ ] File renamed to v1.0
- [ ] Print Audit passed (Module 07)
- [ ] PDF saved to `07_print/`
- [ ] Print manifest completed
- [ ] Cover sheets generated (Module 08)
- [ ] Case sheets generated (Module 09, law only)
- [ ] Binder assembled

## Phase 4 — Exam Preparation

- [ ] Revision pack generated (Module 10)
- [ ] Exam pack generated (Module 11)
- [ ] Exam pack printed and in binder

## Phase 5 — Archive

- [ ] Semester archive completed (Module 12)
- [ ] All files in final organised state
- [ ] Module register marked complete
- [ ] Reuse notes written for future semesters
