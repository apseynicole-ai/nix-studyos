# Standard Operating Procedures — Index

All SOPs follow the same format: trigger → inputs → steps → completion check.

| SOP | Trigger |
|-----|---------|
| SOP-01 Start a New Module | New module added to semester load |
| SOP-02 Add New Lecture Content | New lecture slides or notes received mid-semester |
| SOP-03 Rebuild an HTML Section | A section is wrong enough to rebuild, not just patch |
| SOP-04 Patch an Existing HTML | QA audit or print audit finds specific issues |
| SOP-05 Create Topic Dividers | Cover sheets needed for a module binder |
| SOP-06 Generate Case Sheets | Law module case reference pages needed |
| SOP-07 Prepare for Printing | Hub is at v1.0 and ready to print |
| SOP-08 Final QA Before Printing | Last review before paper goes to printer |
| SOP-09 Semester Rollover | New semester begins, previous hub exists |
| SOP-10 Version Control | Any time a file is about to be modified |
